# AIModal 项目代码重构与优化计划

## 项目概况

AIModal 是一个基于 Tauri 2 的跨平台桌面应用，用于 OpenAI 兼容接口的检测、模型管理、MCP 服务器管理等功能。

**技术栈**: React 18 + TypeScript + Vite 5 + Tailwind CSS 3 + Tauri 2 (Rust)

**核心原则**:
- ✅ 保持现有功能完全不变
- ✅ 渐进式重构，每一步都可独立验证
- ✅ 先创建通用组件，再替换使用点
- ✅ 每个阶段完成后运行测试确保无回归
- ✅ 每个阶段独立 git commit，可随时回退

## 阶段 1: 创建通用 UI 组件库 (P0 最高优先级)

### 目标
在 `src/components/ui/` 下创建一套可复用的基础 UI 组件，统一样式和行为。

### 1.1 目录结构

```
src/components/ui/
├── Button.tsx          # 按钮组件
├── Input.tsx           # 输入框组件
├── Textarea.tsx        # 多行输入框组件
├── Card.tsx            # 卡片容器
├── StatusBadge.tsx     # 状态徽章
├── EmptyState.tsx      # 空状态展示
├── LoadingState.tsx    # 加载状态
├── SectionHeader.tsx   # 区块标题栏
├── KeyValueRow.tsx     # 键值对行
├── index.ts            # 统一导出
└── __tests__/          # 组件测试
    ├── Button.test.tsx
    ├── Input.test.tsx
    └── Card.test.tsx
```

### 1.2 组件详细规格

#### Button.tsx
```tsx
interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'primary' | 'secondary' | 'danger' | 'ghost';
  size?: 'sm' | 'md' | 'lg';
  loading?: boolean;
  icon?: React.ReactNode;
}
```
- 映射到现有 `lib/buttonStyles.ts` 中的样式常量
- 支持 loading 状态自动禁用 + 显示 Loader2 图标
- 所有 `className` 合并使用 `clsx` 或现有方式

#### Input.tsx
```tsx
interface InputProps extends React.InputHTMLAttributes<HTMLInputElement> {
  error?: string;
  icon?: React.ReactNode;
}
```
- 映射到现有 `lib/formStyles.ts` 中的样式常量
- 错误状态显示红色边框 + 错误信息

#### Card.tsx
```tsx
interface CardProps extends React.HTMLAttributes<HTMLDivElement> {
  variant?: 'default' | 'interactive';
  hoverable?: boolean;
}
```
- 统一使用 `bg-surface-card border-border-default rounded-lg` 等 token

#### StatusBadge.tsx
```tsx
interface StatusBadgeProps {
  status: 'success' | 'warning' | 'error' | 'info' | 'unknown';
  label: string;
}
```
- 根据 status 自动选择文字颜色和背景色

#### EmptyState.tsx
```tsx
interface EmptyStateProps {
  icon?: React.ReactNode;
  title: string;
  description?: string;
  action?: React.ReactNode;
}
```

#### LoadingState.tsx
```tsx
interface LoadingStateProps {
  message?: string;
  size?: 'sm' | 'md' | 'lg';
}
```

### 验证点
- `pnpm test` 全部通过
- 新组件可在测试页面正常渲染
- TypeScript 严格模式编译通过

## 阶段 3: 拆分 McpPage.tsx (P1 高优先级)

### 目标
将 `src/components/mcp/McpPage.tsx` 从 ~622+ 行拆分至主组件 ~400 行以内。

### 现状分析
McpPage 已部分模块化，但仍包含大量逻辑。需要进一步拆分：

### 3.1 创建新 hooks

```
src/components/mcp/hooks/
├── useMcpServers.ts      # 服务器 CRUD 逻辑
├── useMcpSync.ts         # 同步逻辑
├── useMcpTest.ts         # 测试逻辑
└── useModelScopeSearch.ts # ModelScope API 搜索
```

每个 hook 职责：
- **useMcpServers**: 服务器列表的增删改查、localStorage/SQLite 持久化
- **useMcpSync**: 同步状态管理、进度追踪、批量同步
- **useMcpTest**: 单个/批量服务器测试、结果收集
- **useModelScopeSearch**: 在线搜索 ModelScope 平台的 MCP 服务器

### 3.2 创建子组件

```
src/components/mcp/components/
├── McpServerList.tsx     # 服务器列表展示 + 操作
├── McpServerEditor.tsx   # 服务器编辑表单
├── McpOnlineSearch.tsx   # 在线搜索面板
├── McpSyncManager.tsx    # 同步管理面板
├── McpBatchTest.tsx      # 批量测试面板
└── index.ts              # 统一导出
```

### 3.3 重构主组件

- 主组件 McpPage.tsx 只保留布局组装和状态协调
- 目标行数: **≤ 400 行**

### 验证点
- `pnpm test` 全部通过 (含 `McpPage.test.tsx`)
- MCP 功能完整测试: 服务器增删改、同步、搜索、测试
- TypeScript 严格模式编译通过

## 风险控制

| 风险 | 缓解措施 |
|------|---------|
| UI 样式变化 | 逐个组件对比视觉差异，保留原有 className |
| 功能回归 | 每个阶段完成后运行 `pnpm test` + 手动测试 |
| 状态管理错误 | 保持现有 hooks 逻辑不变，只拆分 JSX |
| 性能下降 | 使用 React.memo 优化，监控渲染次数 |
| 引入新 bug | 每个阶段独立 git commit，可随时回退 |
